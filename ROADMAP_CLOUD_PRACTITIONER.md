# Roadmap AWS Certified Cloud Practitioner

## Objetivo

Preparar la certificación AWS Certified Cloud Practitioner de forma progresiva.

## Dominios a reforzar

### 1. Cloud Concepts
Qué es la nube, ventajas, elasticidad, escalabilidad, alta disponibilidad, agilidad, pago por uso.

### 2. Security and Compliance
Modelo de responsabilidad compartida, IAM, seguridad básica, cumplimiento y trazabilidad.

### 3. Technology and Services
EC2, S3, RDS, DynamoDB, VPC, Route 53, CloudFront, Lambda, CloudWatch, CloudTrail.

### 4. Billing, Pricing and Support
Facturación, modelos de precio, optimización de costes, soporte y mejores prácticas.

## Ruta de preparación

## Técnica para responder preguntas

1. Identificar qué problema plantea la pregunta
2. Distinguir entre servicio de cómputo, red, almacenamiento, seguridad o base de datos
3. Eliminar opciones claramente incorrectas
4. Elegir la opción más alineada con el caso de uso

## Servicios más preguntables

- Amazon EC2
- Amazon S3
- AWS IAM
- Amazon VPC
- Amazon RDS
- Amazon DynamoDB
- Amazon Route 53
- Amazon CloudWatch
- AWS CloudTrail
- AWS Lambda
- Elastic Load Balancing
- Auto Scaling

## Errores típicos

- confundir escalabilidad con alta disponibilidad
- confundir S3 con EBS o EFS
- confundir RDS con DynamoDB
- pensar que AWS hace toda la seguridad
- no diferenciar monitorización de auditoría

## Meta final

Llegar al examen siendo capaz de:

- reconocer el servicio adecuado
- entender el vocabulario del examen
- responder con lógica, no solo con memoria
